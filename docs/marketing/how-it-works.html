Document AI Form Parser

Document Classification

Gemini Flash classifier

Hybrid RAG Retrieval

Retrieval fuses vector similarity (60% weight) with PostgreSQL FULLTEXT keyword search (40% weight) using Reciprocal Rank Fusion. Neither alone is sufficient for medical records.

pgvector + pg_trgm + RRF

Knowledge Graph — Hebbian Learning

Edge weights between entities decay 5% per 30-day cycle if not reinforced — the graph "forgets" stale connections and strengthens recent ones. Three-tier entity resolution: exact → fuzzy → semantic.

Hebbian neuro-plasticity model
Stage 2

The Claims Workspace

Each claim opens into a 12-tab detail view. Every tab surfaces pre-processed, AI-extracted data alongside the examiner's own notes and actions.

Tab 01
Overview
Claim summary, injured worker info, employer, insurer, injury date, body parts, and current status. Starting point for every interaction.
Tab 02
Documents
All uploaded documents with classification, processing status, extracted fields, and full text. Click any document to view the parsed content.
Tab 03
Deadlines
Full statutory deadline set per claim, each showing consequence of a miss: 1-day DWC-1 provision (LC 5401) · 14-day first TD payment — 10% self-imposed penalty, rising to 15% July 2026 (LC 4650 / SB 1234) · 15-day DWC-1 acknowledgment (10 CCR 2695.5(b)) · 20-day P&S report objection — unfavorable WPI locks in (LC 4061) · 5 business-day prospective UR — missed deadline means treatment is authorized by operation of law (8 CCR 9792.9) · 40-day claim determination — claim presumed compensable if no denial (10 CCR 2695.7(b) / LC 5402(b)) · 30-day delay notice update cycle — each missed letter is a separate audit violation (10 CCR 2695.7(c)).
Tab 04
Coverage
AOE/COE determination per body part. Records industrial vs. non-industrial apportionment, accepted/denied status, and denial rationale.
Tab 05
Medical Billing
Aggregate medical financials: bills received, paid, disputed, and pending. OMFS fee schedule comparison. UR/IMR history.
Tab 06
Benefits
TD rate calculator (AWE × 2/3, statutory min/max), PD rating worksheet, wage loss tracking, and payment schedule with statutory citations.
Tab 07
Liens
Lien tracking: claimant attorney, medical providers, EDD, WCAB. Status: asserted, resolved, waived. Priority ordering.
Tab 08
Investigation
Automated checklist: 20 investigation tasks generated from document analysis. Tracks completion status and flags gaps that increase risk exposure.
Tab 09
Timeline
Chronological event log: every document receipt, payment, communication, and determination with date, actor, and source document.
Tab 10
AI Chat
UPL-filtered chat interface. Green zone: factual Q&A. Yellow zone: statistical data + disclaimer. Red zone: blocked, attorney referral generated.
Tab 11
Audit Trail
Append-only event log for this claim: 60+ event types, IP address, user agent, timestamps. Read-only. 7-year retention enforced.
Tab 12
Education
Contextual regulatory education: Tier 1 dismissable basics, Tier 2 always-present statutory authority. Adapts to the current tab and claim state.

🟢 Green Zone

Factual Response

Delivered with source citations. No disclaimer required. Includes: diagnosis summaries, WPI%, TD calculations, deadline dates, payment status, MTUS guidelines.

🟡 Yellow Zone

Statistical Data + Mandatory Disclaimer

Comparative claim data delivered with mandatory: "This is statistical information, not legal advice. Consult defense counsel for legal guidance."

🔴 Red Zone

Blocked — Attorney Referral

Query blocked. Examiner sees: "This question requires legal analysis. Please contact defense counsel." A counsel referral record is auto-generated.

Stage 6

The Knowledge Graph

AdjudiCLAIMS builds a graph of every entity in every document — injured workers, diagnoses, providers, employers, dates, amounts — and the connections between them. The graph gets smarter over time.

🧩

Entity Extraction

Named entities extracted from every document chunk: people, organizations, medical diagnoses (ICD-10), dates, dollar amounts, body parts, and regulatory references.

3-tier resolution: exact match → fuzzy match (Levenshtein) → semantic match (embedding similarity)
🔗

Edge Weight Learning

Connections between entities are weighted by co-occurrence frequency and recency. The graph learns which relationships matter most for claims outcomes.

Hebbian decay: edge weights decrease 5% per 30-day cycle unless reinforced by new evidence
⚠️

Contradiction Detection

When a new document contradicts prior records — different WPI%, conflicting diagnosis, different injury date — the system flags the contradiction for examiner review.

Contradiction tracking model: source A vs. source B with confidence scores for each
🔍

Graph-Enhanced Retrieval

When an examiner asks a question, the graph expands the retrieval scope — surfacing related entities and documents that a pure vector search would miss.

Graph traversal depth configurable per query type (default: 2-hop)
🏗️

Consolidation Runs

Nightly consolidation merges duplicate entity nodes, decays stale edges, and recomputes community clusters across the full claim graph.

Maintenance window: 02:00–04:00 UTC daily, zero downtime for read operations
📈

Gets Smarter Over Time

The more documents processed, the richer the graph. Long-running claims develop dense entity networks that surface non-obvious connections between events and providers.

G6 Trust system: entity trust scores updated with each reinforced connection
