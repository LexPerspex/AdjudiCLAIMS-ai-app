## Plan Summary

**Approach taken:** Wrap the existing `letter-template.service.ts` and `document-generation.service.ts` infrastructure rather than introducing a parallel write path or a headless-Chromium PDF dependency. Per-payment letters reuse the existing `generateLetter` persistence + audit-logging path. LC 3761 employer notifications are modeled as a discriminated union of structured event types (`BENEFIT_AWARD` and `CLAIM_DECISION`) so additional event types can be added without breaking callers. PDF download is implemented as a thin `Content-Disposition: attachment` wrapper around the existing `/html` print renderer — the user's browser handles the actual PDF artifact via Print → Save as PDF.
