Authentication & Access Control

Authentication Controls

AdjudiCLAIMS uses modern credential security with multi-factor authentication and role-based access enforcement at every API layer.

Control Implementation Standard Status
Password Hashing Argon2id — memory-hard, timing-attack resistant. Configured: 64 MB memory, 3 iterations, 4 parallelism. OWASP PHC Winner Implemented
Multi-Factor Authentication TOTP (Time-based One-Time Password) — RFC 6238 compliant. 30-second window. Backup codes provided on enrollment. NIST SP 800-63B AAL2 Implemented
Account Lockout 5 failed login attempts → 15-minute lockout. Event logged to audit trail with IP address. OWASP A07:2021 Implemented
Session Security HttpOnly, Secure, SameSite=Strict cookies. Session tokens rotated on privilege change. Idle timeout: 30 minutes. OWASP Session Management Implemented
Role-Based Access Control (RBAC) 3 roles: CLAIMS_EXAMINER, CLAIMS_SUPERVISOR, CLAIMS_ADMIN. Enforced at API middleware layer — not just UI. Every route validates role before processing. Principle of Least Privilege Implemented
Password Complexity Minimum 12 characters. Checked against zxcvbn strength estimator (score ≥ 3 required). NIST SP 800-63B Implemented
Credential Storage All secrets (API keys, DB connection strings, session secrets) stored in GCP Secret Manager. Never in environment variables, config files, or git history. CIS Controls v8 #4.7 Implemented
Data Architecture

Multi-Tenant Data Isolation

AdjudiCLAIMS is multi-tenant. Carrier, TPA, and self-insured organizations are isolated at the database query level — not just the application layer.

🏢

Organization Scoping

Every database query includes an organizationId filter. A user from Carrier A cannot retrieve records from Carrier B — even with direct database query construction.

  • Enforced at Prisma query layer
  • Tested in integration test suite
  • SOC 2 CC6.1 compliant
📄

Document Access Levels

Documents have granular access levels: ALL_ROLES (shared), EXAMINER_AND_ABOVE (no admin-only access), SUPERVISOR_AND_ABOVE, and ATTORNEY_ONLY.

  • Attorney work product excluded at query layer
  • ATTORNEY_ONLY documents never returned to examiner roles
  • Data boundary tested in E2E suite
🔐

Encryption

All data encrypted at rest (AES-256) and in transit (TLS 1.3). Database connections require SSL. GCS buckets use customer-managed encryption keys.

  • TLS 1.3 enforced — TLS 1.0/1.1 disabled
  • HSTS headers on all responses
  • Certificate rotation automated via GCP

Infrastructure Security

GCP Infrastructure

AdjudiCLAIMS is deployed exclusively on Google Cloud Platform using managed services with zero infrastructure to patch.

☁️

Cloud Run (API + Workers)

Serverless container execution. Auto-scales to zero under no load. Non-root container user enforced. No persistent VM attack surface.

  • Non-root container user (UID 1000)
  • Read-only root filesystem
  • Separate service accounts per workload
  • CPU/memory limits enforced per service
🗄️

Cloud SQL (PostgreSQL 16)

Managed PostgreSQL with automated backups, point-in-time recovery, and private IP connectivity only. No public internet exposure.

  • Private IP only (no public endpoint)
  • Automated daily backups (7-day retention)
  • PITR enabled (1-minute granularity)
  • SSL required for all connections
🔑

Secret Manager

All credentials stored in GCP Secret Manager with version history and audit logging. Zero hardcoded secrets anywhere in the codebase.

  • IAM-controlled access per service account
  • Automatic secret rotation supported
  • Access logged to Cloud Audit Log
  • Zero secrets in environment variables or git
Anomaly Detection

Sliding-Window Detection Rules

Three automated detection rules monitor for unusual activity patterns. Triggers generate security alerts and audit events.

> 50 docs
Rapid Document Access
More than 50 document accesses within a 5-minute window by a single user triggers a security review flag.
Action: Security alert generated. Audit event logged. Account flagged for review by CLAIMS_ADMIN.
5 attempts
Failed Login Threshold
5 consecutive failed login attempts for an account triggers automatic 15-minute lockout and security alert.
Action: Account locked. IP address logged. Security alert generated. Audit event with IP + user agent.
Off-hours
Bulk Export Detection
Bulk data exports (DSAR, audit log exports) initiated outside business hours (defined per organization timezone) are flagged for review.
Action: Export proceeds but security alert generated. CLAIMS_ADMIN notified. Audit event with export scope and user.
Error Handling & Logging

Production Error Security

Stack traces, database errors, and internal system details are never exposed to end users in production environments.

ControlImplementation
Stack Trace Stripping Production API returns generic error messages. Internal details logged to Sentry only — never to the HTTP response.
No PHI in Error Logs Error logs capture document IDs, claim IDs, and event codes — never document content, patient names, or PHI fields.
Sentry Integration Errors reported to Sentry with scrubbed context. PHI scrubbing rules configured to strip sensitive fields from Sentry payloads.
Database Error Handling Prisma errors are caught and mapped to generic HTTP responses. Raw PostgreSQL error messages are never returned to clients.
Rate Limiting API endpoints rate-limited per user session. Authentication endpoints have stricter limits (5 requests/minute). Chat endpoints: 30 requests/minute.

Data Retention & Deletion

Retention Policy

AdjudiCLAIMS enforces a 7-year retention policy consistent with California Workers' Compensation record requirements. Some records are immutable by design.

Data Type Retention Period After Retention Mutable?
Claim records 7 years from claim close Purged from active database. Archived to GCS cold storage. Examiners can update. Audit trail is immutable.
Uploaded documents 7 years from upload Deleted from GCS and document table. Embeddings purged. No modifications to original documents. Metadata editable.
Audit events 7 years from event (minimum) Archived to GCS cold storage. Never deleted during retention period. Never — append-only. No UPDATE or DELETE permitted.
Chat messages 7 years from message date Purged with parent claim. No — messages are append-only once sent.
User accounts Retained while org active Soft-deleted on account closure. 30-day grace period before hard delete. Users can update profile. Audit trail immutable.
AI embeddings / vectors Tied to parent document Purged when parent document is purged. Regenerated on model update. Original not preserved.
Right to Deletion (CCPA): Data Subject Access Requests and right-to-deletion requests are handled via dedicated API endpoints. Claims data linked to open WCAB proceedings cannot be deleted until proceedings close — this restriction is disclosed at time of request. Response time: 30 calendar days per Cal. Civ. Code § 1798.105(c).
