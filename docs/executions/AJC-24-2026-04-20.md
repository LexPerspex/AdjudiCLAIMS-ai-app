## What Was Delivered

Next.js 15 + React 19 + Tailwind v4 frontend at `packages/insurance-claims-case-generator/frontend/`, matching the `merus-test-data-generator` pattern.

### Pages

| Route | Description |
|-------|-------------|
| `/` | Scenario Selector — 13 scenario cards in a responsive grid, each showing display name, flag badges (Litigated, CT, Psych, PTD, Denied, Death, Expedited, etc.), doc count range, and a Generate button that navigates to `/generate?scenario=<slug>` |
| `/generate` | Generate Form — scenario dropdown (pre-populated from URL param), optional seed input, PDF toggle, AdjudiCLAIMS seed toggle (env selector local/staging + base URL + credentials); calls `POST /api/v1/generate`, navigates to `/jobs/<case_id>?sync=1` |
| `/jobs/[id]` | Job Status Poller — polls `GET /api/v1/jobs/{id}` every 2s, progress bar (0–100%), status text, document count summary; on complete shows "Download ZIP" button (`GET /api/v1/export/{id}`). Also handles sync=1 case from single-case generate |
| `/batch` | Batch Page — total count slider (1–500), scenario distribution table (13 rows, weight input per scenario, auto-normalizes to 100%), case counts column; calls `POST /api/v1/batch`, navigates to `/jobs/<job_id>` |
