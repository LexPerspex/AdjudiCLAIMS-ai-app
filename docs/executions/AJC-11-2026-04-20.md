## Audit Findings

**Pre-existing gaps filled:**
1. Document upload flow — no dedicated spec existed
2. Document OCR status classification display — not tested
3. Calculator numeric output validation — only a loose HTML-string check existed
4. Compliance dashboard detail — only page-load presence checks existed
5. MTUS guideline lookup — zero E2E coverage

**Design decisions:**
- Matched existing defensive pattern: `test.skip()` when element not visible — CI-safe without live DB
- No POM (Page Object Model) — matches existing flat pattern
- No hardcoded credentials — `E2E_EXAMINER_EMAIL`/`E2E_EXAMINER_PASSWORD` env vars with test-only fallbacks
- No MSW/fixtures — tests run against `DEPLOYMENT_URL` (Cloud Run)
- UPL prohibition negative assertion in `mtus.spec.ts` — verifies MTUS content does not contain prohibited legal conclusions
