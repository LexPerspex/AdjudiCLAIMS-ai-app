**Key decisions:**
- Data lives in `server/data/`, not the service file — matches existing codebase pattern (`regulatory-kb.ts` etc.) and keeps the service file focused on logic
- Service API contracts preserved exactly — `matchMtusGuidelines`, `matchMtusGuidelinesFromKb`, `getGuidelineDetail` signatures and return shapes unchanged. No callers needed updates
- `isStubData=true` semantics retained for backward compat — now means "served from bundled data" rather than "synthetic placeholder", documented in the JSDoc
- Body-part resolution: exact alias match → partial substring match → CPT fallback. Partial match enables natural examiner phrasings ("low back pain", "left knee", etc.) without exhaustive enumeration
- Frontend `_app.mtus.tsx` (uses different `/mtus/search` endpoint) intentionally out of scope — ticket specifically targets the MTUS knowledge base, not the search UI
