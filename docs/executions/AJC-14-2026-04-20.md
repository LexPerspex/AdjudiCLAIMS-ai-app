## Plan Summary

**Approach taken:** The three G5 Trust UX frontend components (`ConfidenceBadge`, `EntityPanel`, `ProvenanceCard`) already existed in `app/components/graph/` but were not wired into the chat pipeline. The implementation added `GraphTrustData` types to the backend service, captured the graph query result (previously discarded after formatting), built a transparent `buildGraphTrustData()` function that derives per-message trust data from the graph traversal result and RAG citations, serialized it through the API route, extended the frontend `ChatMessage` type, and wired the new `EntitySummarySection` and `GraphProvenanceSection` components into `AssistantMessage` with appropriate UPL zone guards.

**Key decisions:**
- `buildGraphTrustData()` is a pure function isolated from the UPL enforcement pipeline — reads already-filtered graph data so access control guarantees are unchanged
- `graphTrust` is optional on frontend `ChatMessage` for backward compatibility with persisted message history
- Trust UX hidden (`!isRed` guards) for RED zone responses — confidence display is factual/GREEN zone only
- Overall confidence = mean of graph node confidences > mean of citation similarities > 0.5 neutral fallback
