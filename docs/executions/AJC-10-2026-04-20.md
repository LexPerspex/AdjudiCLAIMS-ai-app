## Summary

## Audit: Current State (Pre-Implementation)

**Already enforced:**

- Organization-scoped claim access: `verifyClaimAccess` middleware filters by `organizationId` for all claim-related routes (documents, deadlines, investigation, coverage, medical-billing, liens, letters, referrals, workflows, reports, chat).
- Document access filtering: `document-access.service.ts` enforces UPL data boundary via `getDocumentAccessFilter` (excludes `ATTORNEY_ONLY`, `containsLegalAnalysis`, `containsWorkProduct`, `containsPrivileged`) applied to document list and individual document fetch.
- RAG retrieval filter: `getRagAccessFilter` excludes restricted documents from DocumentChunk queries before content reaches the LLM.
- KB filtering: `kb-access.service.ts` blocks `pdrs_2005` and `crpc` sources and `legal_principle`/`case_summary`/`irac_brief` content types for all examiner roles.
- Chat isolation: Chat routes use `verifyClaimAccess` — sessions scoped to org-owned claims.
- Organization routes: `GET /orgs/:id` and members endpoint check `user.organizationId === id`.
- 31 existing data-boundary tests passing.

**Gaps identified:**

1. No cross-org isolation tests — no tests explicitly proved Org B cannot access Org A data via the API.
2. Soft-delete bypass — `verifyClaimAccess` did not check `deletedAt`, allowing soft-deleted claims to be accessed if the caller knew the ID.
3. Inline `GET /claims/:id` and `PATCH /claims/:id` routes did not check `deletedAt`.
4. Soft-deleted documents accessible via `GET /documents/:id` (no `deletedAt` guard).
5. No centralized documentation of the cross-org boundary enforcement architecture.

## Changes Made

### `server/middleware/claim-access.ts`

- Added `deletedAt: true` to Prisma select.
- Added soft-delete guard: `claim.deletedAt != null` → returns `{ authorized: false, claim: null }`.
- Treats soft-deleted claims the same as non-existent (404 upstream).

### `server/routes/claims.ts`

- `GET /api/claims/:id`: Added `deletedAt` to select, added soft-delete guard before org check.
- `GET /api/claims/:id`: Strips `deletedAt` from response (not part of public API contract).
- `PATCH /api/claims/:id`: Added `deletedAt` to select, added soft-delete guard before org check.

### `server/routes/documents.ts`

- `GET /api/documents/:id`: Added `deletedAt` to select, added soft-delete guard (`document.deletedAt != null` → 404).

### `server/services/org-boundary.service.ts` (new)

- `isSameOrg(resourceOrgId, userOrgId)`: Null-safe org ID comparison utility.
- `orgScope(orgId)`: Prisma where-clause fragment that scopes top-level queries to an org and excludes soft-deleted records.
- Full JSDoc documenting all 5 enforcement layers (session, claim, document, KB, organization).

### `tests/unit/claim-access.test.ts`

- Added 4 new soft-delete guard tests (examiner/supervisor/admin blocked, non-deleted passes).
- Updated all mock return values to include `deletedAt: null`.
- Updated Prisma query assertion to expect `deletedAt: true` in select.
- Changed `.toEqual` to `.toMatchObject` for partial matching where `deletedAt` is an implementation detail.

### `tests/unit/org-boundary.test.ts` (new)

- 11 pure service tests for `isSameOrg` and `orgScope`.
- 14 route integration tests for cross-org isolation:
  - `GET /claims/:id`: Org 2 examiner, supervisor, admin all receive 404.
  - `PATCH /claims/:id`: Org 2 examiner, supervisor both receive 404.
  - `GET /claims/:claimId/documents`: Org 2 examiner, supervisor receive 404.
  - `GET /documents/:id`: Org 2 examiner, supervisor, admin receive 404.
  - `POST /claims/:claimId/documents`: Org 2 examiner receives 404.
  - `GET /claims/:claimId/timeline`: Org 2 examiner receives 404.
  - Soft-deleted claim in same org: Org 1 examiner receives 404.
  - Soft-deleted document in same org: Org 1 examiner receives 404.

### `tests/unit/data-boundaries.test.ts`

- Updated `MOCK_CLAIM` to include `deletedAt: null` (required by updated `verifyClaimAccess`).
- Updated `MOCK_DOCUMENT_EXAMINER` to include `deletedAt: null` (required by updated document route).
